# RAMA Project 

This project develops a comprehensive data engineering workflow for historical data generated by the **Automatic Atmospheric Monitoring Network (RAMA)**, part of the **Atmospheric Monitoring System of Mexico City (SIMAT)**.

The main objective is to build a reproducible pipeline capable of extracting, storing, cleaning, transforming, integrating, and documenting air quality data published by RAMA, resulting in a structured and reliable dataset for environmental research, statistical analysis, and data science applications.

---

## 🎯 Objectives

- Automate the acquisition of historical RAMA datasets.
- Preserve original source data through a layered data architecture.
- Standardize and document information from multiple pollutants and monitoring stations.
- Detect and handle missing values and data inconsistencies.
- Integrate historical records into consolidated datasets.
- Generate outputs in open and efficient formats for analytical workloads.

### DATA Layers

#### LANDING

Stores the original ZIP files downloaded directly from the official RAMA source. This layer preserves raw data exactly as published and provides full traceability.

#### RAW

Contains extracted XLS and converted CSV files. Data remains unmodified and serves as the starting point for cleaning and transformation processes.

#### PROCESSED

Stores cleaned, transformed, validated, and integrated datasets ready for analysis, visualization, machine learning, or database storage.

## 🔍 Key Findings from the Ingestion Phase

- 16 years of historical data downloaded (2011–2026)
- 144 XLS files successfully extracted
- 144 CSV files generated
- 9 pollutants consistently available across the study period
- Hourly measurements identified
- Missing values represented using the code `-99`
- Monitoring station coverage varies by pollutant
- File integrity and consistency successfully validated
